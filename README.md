# HW1: Coding Assistant Onboarding

## What's This Repo?
This is my homework assignment showing how I used Codex (GitHub Copilot) as an AI coding assistant. I generated some content with AI, then edited it myself to make it actually reflect my interests. The topic ended up being about healthcare access in rural and developing countries, which is something I concern about.

## How I Used Git

The workflow I went through:

1. Created an empty repo on GitHub (wanted my first commit to have all three files like the assignment said)
2. Cloned it to my computer using GitHub Desktop
3. Made a first commit with README.md, assistant_draft.md, and final_version.md all together
4. Used Codex/Copilot to write the content for assistant_draft.md
5. Copied that file and manually edited it to create final_version.md with my personal details
6. Updated this README
7. Will add my video link here once I record it

**Git stuff I used:**
- `git clone` - to get the repo on my computer
- `git add` - to stage files before committing
- `git commit -m "message"` - to save my changes
- `git push` - to upload everything to GitHub

## What I Learned

**Git stuff:**
- Making sure my first commit had everything I needed
- Writing commit messages that actually make sense later
- GitHub Desktop is pretty easy to use once you figure it out

**AI assistant stuff:**
- Codex is good at generating a decent first draft fast
- But the prompt matters a lot - I had to be specific about what I wanted
- The AI output felt generic until I went in and added my own details
- Definitely can't just accept whatever the AI gives you without editing
